Technology Dependency Map

Dependency Reference

Strategic Purpose

A reference map of the interdependency relationships between SBSP technology subsystems, organised into dependency tiers that show which technologies must reach readiness before others can be meaningfully developed at SBSP-relevant scale. The map identifies critical path constraints — the sequencing of technology development that determines the minimum time to system integration.

Designed for: technology programme managers, researchers, government technology advisors, institutional investors

Decision value: Technology development cannot be optimally sequenced without understanding which subsystems are prerequisites for others. The dependency map prevents the common error of funding parallel development across all subsystems while the critical path bottlenecks remain unresolved. It identifies which technology investments unlock the most subsequent development steps.

Reference Data

Tier Description Components Current Status Unlocks
Tier 1 — Foundation Technologies (No internal SBSP dependencies) Technologies that must reach sufficient readiness before SBSP-specific development can proceed at scale. Progress in these technologies is necessary but not sufficient for SBSP viability. Launch vehicle economics (heavy-lift cost reduction); Orbital logistics infrastructure (refuelling, servicing); Space-grade power electronics manufacturing scale; Spectrum regulatory framework for space-to-ground microwave transmission Launch economics improving but not at SBSP-required threshold. Orbital servicing in early development. Space power electronics improving. Spectrum regulatory framework does not currently exist for operational SBSP. System integration testing at meaningful scale; economic viability modelling with confirmed assumptions
Tier 2 — Core Subsystem Technologies (Depend on Tier 1) The primary functional subsystems of SBSP that must each reach sufficient TRL before integrated system testing is feasible. Each has specific Tier 1 dependencies. Solar collection array at SBSP tile scale; Phased array WPT transmitter at required power density; In-space structural assembly system; Rectenna ground receiver at grid-connection scale Solar collection: heritage components mature, SBSP tile integration at TRL 3-4. WPT transmitter: component demonstrated (TRL 4-5), operational-scale undemonstrated. In-space assembly: conceptual for SBSP scale (TRL 2-3). Rectenna: components tested, grid-scale not built (TRL 5-6). Subsystem integration testing; demonstration satellite development; intermediate-scale system prototyping
Tier 3 — Integration Technologies (Depend on Tier 2) Technologies that integrate Tier 2 subsystems into a functional system. These cannot be meaningfully developed until the subsystems they integrate reach sufficient readiness. Beam control and steering across operational GEO-to-ground path; Thermal management for combined solar collection and WPT operation; Power conditioning chain from array DC through microwave to rectenna DC; System operations and fault detection at orbital scale Beam control: component-demonstrated in laboratory and MAPLE; operational-range steering not demonstrated. Thermal management: modelled at system level, not tested at SBSP scale. Power conditioning: components mature in isolation, integrated chain efficiency at SBSP scale not demonstrated. Operations: simulation and planning only. End-to-end system testing; pilot demonstration programme
Tier 4 — Deployment-Prerequisite Technologies (Depend on Tier 3) Technologies and capabilities that must be in place before deployment-scale systems can be operated. These are the final layer before grid-connected SBSP becomes operationally feasible. End-to-end system performance validation across full transmission chain; Grid integration infrastructure and regulatory approval; Safety certification for microwave transmission footprint; Commercial framework for SBSP power purchase or infrastructure investment End-to-end system: not yet attempted even at small scale. Grid integration: regulatory and infrastructure framework does not exist. Safety certification: frameworks being studied; operational precedent does not exist. Commercial framework: no SBSP power purchase agreement has been executed. Deployment-scale investment decision; operational SBSP system

How to Read This Tool

Read the map from Tier 1 upward. A programme that is advancing Tier 3 technologies while Tier 1 prerequisites remain unresolved is advancing in a context where the enabling foundations are not yet confirmed. The critical path bottleneck is the Tier 1 technology with the longest remaining development runway — currently launch economics and spectrum regulatory framework. Investment in Tier 2 and Tier 3 technologies is not wasted (component TRL development accumulates), but system integration cannot proceed until the lowest Tier 1 component is sufficiently mature.

Methodology Boundary

Dependency relationships are derived from systems engineering analysis of SBSP architecture as documented in major feasibility studies. The tier structure and dependency arrows represent editorial synthesis of multiple source documents, not a single authoritative systems engineering breakdown. Dependency relationships represent technical logic (A must work before B can be tested meaningfully) rather than organisational or funding sequence.

Claim Boundary

This dependency map documents technical dependency logic for SBSP system development. It does not claim that resolving critical path dependencies will lead to deployment, that any specific timeline applies, or that the dependency structure is complete. Novel failure modes at the intersection of subsystems may introduce additional dependencies not captured here.
