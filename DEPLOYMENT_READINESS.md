**Status:** Approved  
**Phase:** deployment_readiness_v1

## Purpose

This document governs how the generated `public/` output is deployed to production. It defines the deployment pipeline, its constraints, and the decisions that must not be reversed without a documented [S1-DL-*] decision.

---

## Deployment Architecture

### Source of Truth

The source of truth for all content is:

```
main/data/           — page registry, seed data, link graph
main/content/        — authored page content
main/scripts/        — build engine and validators
main/templates/      — (future) template layer
governance docs      — DECISION_LOG, QUALITY_GATE.md, etc.
```

`public/` is the build output — not the source.

### Build and Deployment Path

```
push to main
  → GitHub Actions: quality-gate.yml (full validation)
  → GitHub Actions: deploy-pages.yml (build + deploy)
      → validate (full pipeline)
      → build.py → public/
      → upload-pages-artifact
      → deploy to GitHub Pages
```

No manual uploads. No direct edits to `public/`. No deployment without passing the quality gate.

---

## Governance Rules

### What MUST be true at all times:

1. **No deployment without quality gate pass.** The deploy workflow runs the full validator stack before build. The build never runs unless all pre-build validators pass. No exceptions.

2. **GitHub Pages source = GitHub Actions.** The Pages source must be set to `GitHub Actions` in repository Settings → Pages. Direct branch deployment is not allowed.

3. **No Cloudflare API token in this workflow.** Cloudflare is used later as a DNS, TLS, and security layer only. It does not participate in the build or deployment pipeline.

4. **Canonical URLs use `https://space-based-solar-power.com/`.** The domain is set in `main/scripts/build.py` as `DOMAIN = "https://space-based-solar-power.com"`. This value must not be changed to a GitHub Pages subdomain.

5. **`robots.txt` points to the correct sitemap.** Generated by `build.py`. Sitemap reference is derived from the build output, not hardcoded to a GitHub subdomain.

6. **Sitemap contains only validated public URLs.** `validate_sitemap.py` enforces this. No sitemap URL may be added without an `approved_for_launch` + `sitemap: true` page entry.

7. **`.nojekyll` must exist inside `public/`.** Required to prevent GitHub Pages from running Jekyll on the output directory. Generated by `build.py`.

## GitHub Pages Configuration

### Repository Settings

- Settings → Pages → Build and deployment → Source: **GitHub Actions**
- Custom domain: `space-based-solar-power.com` (set via repository settings, not CNAME file)
- Enforce HTTPS: enabled

### Workflow File

`.github/workflows/deploy-pages.yml`

Triggers on: push to `main` only.  
Requires: `pages: write` + `id-token: write` permissions.  
Job structure: `build` job (validates + builds + uploads artifact) → `deploy` job (deploys artifact).

## Tracked public/ Policy

During Publication Trial v1A, `public/` is tracked in the repository for auditability (see [S1-DL-012]).

Once the deployment pipeline is fully automated and stable:
- A new decision entry [S1-DL-*] must document the transition
- `.gitignore` must be updated to exclude `public/`
- The existing tracked `public/` must be removed from history via `git rm --cached`

This transition must not happen silently.

---

## Node.js Warning in GitHub Actions

The GitHub Actions runner currently shows deprecation warnings about Node.js versions used internally by some action versions. These are:

- Not build failures
- Not deployment failures
- Not quality gate failures

They will be resolved by upgrading action versions in the future. This is tracked separately and must not block the deployment pipeline.

## Deployment Acceptance Criteria

Before declaring Deployment Readiness v1 complete:

- [ ] `.github/workflows/deploy-pages.yml` merged to `main`
- [ ] Repository Settings → Pages → Source = GitHub Actions
- [ ] First successful deploy workflow run on `main`
- [ ] `https://space-based-solar-power.com/` serves the generated output
- [ ] Canonical URLs in HTML match `https://space-based-solar-power.com/`
- [ ] `robots.txt` accessible at root
- [ ] `sitemap_index.xml` accessible at root
- [ ] `.nojekyll` present in deployed output
- [ ] No Jekyll processing artifacts in deployed pages
