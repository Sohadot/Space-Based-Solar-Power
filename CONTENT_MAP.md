# Content Map: Space-Based-Solar-Power.com

## 1. Purpose

This document defines the approved content map for Space-Based-Solar-Power.com.

The content map translates the asset thesis, project doctrine, multilingual policy, source policy, interface thesis, design system, and content strategy into an operational publication structure.

The project is not building a normal website.

It is building a sovereign-grade, multilingual, source-disciplined, interface-driven reference asset for Space-Based Solar Power and Orbital Energy Infrastructure.

This content map exists to prevent:

- random page creation;
- thin SEO expansion;
- placeholder pages;
- broken internal links;
- fake authority;
- unreviewed localized content;
- unsourced technical claims;
- premature tools;
- disconnected articles;
- weak buyer logic.

Every public route must have a strategic role.

---

## 2. Governing Content Rule

No public page may exist merely because it is expected on a normal website.

Every page must strengthen at least one of the following:

- category authority;
- conceptual story;
- contextual framework;
- source trust;
- SEO quality;
- internal-link structure;
- strategic tools layer;
- multilingual authority;
- buyer logic;
- future monetization;
- acquisition readiness.

If a page does not strengthen one of these layers, it should not be published.

---

## 3. Publication Status Definitions

The content system uses the following route statuses.

```text
approved_for_launch
```

The page is approved for Sovereign Reference Launch v1 and may enter production after content, metadata, links, and validation are complete.

```text
foundation_required
```

The page is part of the core foundation and must be built before deep expansion.

```text
expansion_cluster_1
```

The page belongs to the first reference expansion cluster after the launch foundation.

```text
expansion_cluster_2
```

The page belongs to the second expansion cluster, usually tools, glossary, and question pages.

```text
expansion_cluster_3
```

The page belongs to the third expansion cluster, usually audience, comparison, and regional pages.

```text
future
```

The page is strategically valid but not approved for early production.

```text
blocked
```

The page must not be published until a documented decision changes its status.

## 4. Sitemap and Navigation Rules

### Sitemap Rule

A page may enter `sitemap.xml` only if it is:

- complete;
- canonical-valid;
- metadata-complete;
- internally linked;
- source-disciplined where required;
- free of placeholder content;
- quality-gate approved.

Draft, planned, incomplete, or unreviewed pages must not enter the sitemap.

### Navigation Rule

Public navigation must only expose approved and live routes.

Planned pages may exist in `CONTENT_MAP.md`, `pages.json`, or documentation, but must not appear in public navigation until generated and validated.

### Language Switcher Rule

The language switcher may expose only published and validated localized variants.

It must not link to:

- empty language directories;
- draft translations;
- machine-translated review pages;
- unpublished language routes;
- placeholder localized pages.
